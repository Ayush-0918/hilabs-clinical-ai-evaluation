# Clinical AI Evaluation Framework

This repository implements an evaluation framework for assessing the reliability of a clinical NLP pipeline that extracts structured medical entities from clinical documents.

The framework analyzes outputs generated by a clinical AI system and computes multiple evaluation metrics to identify weaknesses in entity extraction, temporal reasoning, and subject attribution.

The goal is to build a **reliability layer** for clinical AI pipelines that helps detect extraction errors and improve the trustworthiness of structured clinical data.

---

# Clinical AI Pipeline Overview

A typical clinical AI pipeline converts unstructured medical charts into structured medical entities.

Medical Charts (PDF / Scanned Documents)  
↓  
OCR Document Digitization  
↓  
Structured Text (Markdown)  
↓  
Clinical NLP Entity Extraction  
↓  
Structured Medical Entities  
↓  
Evaluation & Reliability Analysis

This project focuses on the **evaluation stage**, where extracted entities are analyzed to measure the performance and reliability of the pipeline.

# Evaluation Metrics

The framework computes several reliability metrics:

### Entity Type Error Rate
Measures incorrect classification of medical entities such as medications, problems, procedures, and tests.

### Assertion Error Rate
Evaluates whether entities are correctly labeled as **POSITIVE**, **NEGATIVE**, or **UNCERTAIN**.

### Temporality Error Rate
Measures the accuracy of detecting whether a medical event is:

- CURRENT
- CLINICAL_HISTORY
- UPCOMING

### Subject Attribution Error Rate
Checks whether the entity correctly refers to the **PATIENT** or a **FAMILY_MEMBER**.

### Event Date Accuracy
Measures the accuracy of extracted clinical event dates.

### Attribute Completeness
Evaluates whether additional attributes such as medication dosage, test values, or measurement units are extracted.
