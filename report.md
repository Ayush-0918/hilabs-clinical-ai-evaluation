This project focuses on the **final stage**, where extracted entities are evaluated to measure the reliability of the system.

---

# Dataset Overview

The dataset contains outputs generated by a clinical AI pipeline processing medical charts.

Each dataset folder contains:

- A **Markdown file (.md)** containing OCR-processed medical text
- A **JSON file** containing extracted structured medical entities

The dataset consists of approximately **30 clinical chart folders**, each containing multiple extracted medical entities with associated metadata.

Each entity record contains attributes such as:

- entity
- entity_type
- assertion
- temporality
- subject
- metadata_from_qa

These attributes form the basis for evaluating the reliability of the extraction pipeline.

---

# Approach

The evaluation framework analyzes extracted clinical entities and computes several reliability metrics.

Rather than retraining the model, the approach focuses on **analyzing AI outputs to identify failure patterns**.

The evaluation pipeline consists of the following steps.

---

## 1. Data Ingestion

The framework reads the JSON file from each clinical chart folder and loads all extracted entities for evaluation.

Each entity record is processed individually to analyze its attributes.

---

## 2. Entity Attribute Analysis

Each entity contains several structured attributes:

- Entity text
- Entity type
- Assertion status
- Temporality information
- Subject attribution
- Additional metadata attributes

These attributes are analyzed to detect inconsistencies and incomplete information.

---

## 3. Metric Computation

The evaluation framework computes multiple reliability metrics.

### Entity Type Error Rate

Measures how frequently entities are incorrectly classified into categories such as:

- MEDICINE
- PROBLEM
- PROCEDURE
- TEST
- VITAL_NAME

---

### Assertion Error Rate

Evaluates whether clinical entities are correctly labeled as:

- POSITIVE
- NEGATIVE
- UNCERTAIN

Incorrect assertion detection may lead to misinterpretation of patient conditions.

---

### Temporality Error Rate

Measures the accuracy of identifying temporal context:

- CURRENT conditions
- CLINICAL_HISTORY (past conditions)
- UPCOMING medical events

Temporal reasoning is critical for understanding patient timelines.

---

### Subject Attribution Error Rate

Determines whether clinical entities are correctly associated with:

- PATIENT
- FAMILY_MEMBER

Incorrect attribution may lead to inaccurate patient records.

---

### Event Date Accuracy

Evaluates how accurately event dates are extracted from clinical text.

---

### Attribute Completeness

Measures whether additional structured attributes such as:

- medication dosage
- test values
- measurement units
- administration routes

are successfully extracted.

# Key System Weaknesses

The evaluation revealed several weaknesses in the clinical AI pipeline.

### 1. Uncertain Assertions

Some clinical entities are labeled as UNCERTAIN due to ambiguous language in clinical narratives.

---

### 2. Temporal Reasoning Challenges

Clinical notes often contain complex references to historical and current medical events, making temporality classification difficult.

---

### 3. Metadata Incompleteness

Many entities have empty metadata fields, reducing attribute completeness and limiting structured analysis.

---

# Proposed Reliability Guardrails

To improve reliability of clinical AI pipelines, several guardrails can be implemented.

1. **Temporal rule validation**

   Implement rule-based checks to improve classification of historical vs current conditions.

2. **Metadata completeness validation**

   Ensure medication records include dosage, frequency, and administration route.

3. **Subject attribution validation**

   Improve detection of family history mentions.

4. **Clinical vocabulary validation**

   Validate extracted entities using medical dictionaries.

# Impact

Reliable extraction of clinical entities is essential for building trustworthy healthcare AI systems.

This evaluation framework helps:

# Conclusion

This project presents a **rule-based evaluation framework** for analyzing the reliability of a clinical AI pipeline.

By computing structured evaluation metrics and analyzing error patterns across multiple reasoning dimensions, the framework helps identify weaknesses in entity extraction and clinical reasoning.

Such evaluation layers are critical for improving the transparency, reliability, and safety of AI systems used in healthcare applications.
